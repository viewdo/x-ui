Routing

Content routing is a declarative approach to the features of a multi-page using a single HTML file. Once configured, the browser URL will determine what content is fetched and rendered. You can choose between 'hash' or 'browser' paths. Hash-paths require no extra server-configuration where the browser-mode, does need a rewriting proxy.

How does it work?

By wrapping content in an <x-view> or <x-view-do> elements and setting the url, the contained-content won't be displayed until the URL path matches the path set in the url attribute.

These elements work very similarly to typical web frameworks, minus the framework. You can organize HTML fragments as separate files where it makes sense.
