Routing

Url routing adds URL-based navigation to your single-page's content.

Once configured, the browser URL will determine what content is rendered, allowing users to bookmark and link to multiple "pages", managed as a single file.

How it works

By wrapping content in an <x-app-view> element and setting the url attribute, the contained-content won't be displayed until the URL path matches the path set in the url attribute.

Parent routes can have content shared for child routes, like a child-menu or sub-nav.
